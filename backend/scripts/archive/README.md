# 归档临时脚本

本目录存放 `backend/` 根目录下历史遗留的临时脚本（temporary / ad-hoc scripts）。

## 说明

- 这些脚本是项目开发过程中产生的临时性工具代码，包括：
  - 以 `_` 开头的调试 / 验证脚本（如 `_check_*.py`、`_crawl_*.py`、`_test_*.py`、`_sim_*.py`、`_db_*.py` 等）
  - 一次性数据修复脚本（如 `fix_*.py`）
  - 临时爬虫脚本（如 `zhihu_*.py`、`_crawl_*.py`）
  - 临时测试脚本（如 `test_constraint.py`）
- **这些脚本已归档，不再维护**，仅保留作为历史参考。
- 不应在新代码中引用这些脚本，也不应将其纳入生产构建 / CI 流程。
- 如需重新启用某个脚本，请将其迁移至 `backend/scripts/`（非 archive 子目录）并补充必要的文档与测试。
