# 考研模块浏览器级 E2E 走查报告

> 走查时间：2026-07-04  
> 环境：后端 `http://localhost:8000`，前端 `http://localhost:3001`（后因鉴权 fallback 到 `http://localhost:3000`）  
> 执行方式：真实浏览器渲染 + 截图 + 控制台/网络监控

## 总体结论

当前考研模块**数据层与 API 层已打通**，但**前端访问路径存在 P0 级阻断**：

1. 所有 `/kaoyan/*` 子路由受 `AuthGuard` 保护，未登录即重定向到 `/login`。
2. 已登录环境下，子路由 page chunk 加载失败（`ERR_ABORTED`），页面卡在“正在校验登录状态…”。
3. `/kaoyan/community/qa` 与 `/kaoyan/community/posts` 列表路由缺失，返回 404。

在修复上述阻断问题后，仍需补充数据、替换硬编码、优化 AI 等待体验，才能达到“成熟产品”标准。

## 问题清单

### P0 阻断

#### 1. 考研模块未登录不可用
- **复现路径**：浏览器无痕模式访问 `http://localhost:3001/kaoyan/schools` → 自动跳转到 `/login`。
- **影响**：考研院校、导师、备考策略、社区等核心信息无法被搜索引擎或未注册用户体验，严重违背“打破信息差”目标。
- **修复建议**：
  - 在 `AuthGuard` 中将 `/kaoyan` 及其子路由加入白名单，允许匿名浏览。
  - 仅对需要写入的操作（提交导师评价、发布问答/经验贴、AI 选校推荐）要求登录。

#### 2. 已登录子路由卡在“正在校验登录状态…”
- **复现路径**：在已登录的 `localhost:3000` 实例中访问 `/kaoyan/schools` 等子路由，页面只显示 spinner，对应 page chunk `ERR_ABORTED`。
- **影响**：已登录用户也无法使用考研模块。
- **修复建议**：
  - 重启/确认当前代码编译通过；排查 `@vite/client` 与 Next.js dev server 的冲突。
  - 在 `AuthGuard` 中增加校验超时与错误兜底，避免无限 spinner。

#### 3. 社区子路由 404
- **复现路径**：直接访问 `/kaoyan/community/qa` 或 `/kaoyan/community/posts`。
- **影响**：社区入口的独立链接无效，SEO 与导航体验差。
- **修复建议**：
  - 补齐 `app/(app)/kaoyan/community/qa/page.tsx` 与 `app/(app)/kaoyan/community/posts/page.tsx`；
  - 或统一使用 `/kaoyan/community?tab=qa` 并在路由层做重定向。

### P1 体验

#### 4. 首页“平台数据”硬编码
- **位置**：`app/(app)/kaoyan/page.tsx`
- **现象**：“院校数据 100+”“导师信息 44”“真实评价 20+”“活跃用户 500+”为写死数组。
- **修复建议**：调用后端统计接口（如新增 `/api/grad-intel/stats`）动态渲染；无数据时显示骨架屏。

#### 5. 备考策略页硬编码内容
- **位置**：`app/(app)/kaoyan/strategy/page.tsx`
- **现象**：
  - “今日任务”写死为 `背单词 100 个 / 数学真题 2 套 / 专业课笔记整理`。
  - 薄弱诊断中政治/英语/数学均显示“待导入成绩数据”。
  - 备考时间线为固定 3–12 月阶段。
- **修复建议**：
  - 未绑定成绩/测评时显示引导："导入模考成绩" / "完成测评"。
  - AI 选校推荐区改为数据驱动，加载中显示骨架屏，错误时降级展示历史分数线推荐。

#### 6. 导师详情页无雷达图
- **位置**：`app/(app)/kaoyan/mentors/[id]/page.tsx`
- **现象**：仅展示 6 项评分进度条，未实现雷达图。
- **修复建议**：使用 Recharts 或现有 `charts.tsx` 封装雷达图，或在入口文案中移除“雷达图”表述。

#### 7. 列表页错误处理静默
- **位置**：`mentors/page.tsx`、`strategy/page.tsx` 等
- **现象**：`catch` 块为空或仅 `// silent`，用户看不到加载失败提示。
- **修复建议**：统一使用 toast + 空状态组件提示用户。

### P2 优化

#### 8. 默认英文 404 页
- **复现路径**：访问未定义路由。
- **修复建议**：自定义 `app/not-found.tsx`，提供中文文案与“返回考研首页”按钮。

#### 9. 导师列表筛选选项写死
- **位置**：`app/(app)/kaoyan/mentors/page.tsx`
- **现象**：院校/院系下拉为硬编码 10 所学校与 6 个方向。
- **修复建议**：从后端拉取可选院校/院系聚合，或改为可输入搜索。

#### 10. 开发环境 Vite/Next.js 配置冲突
- **现象**：控制台反复请求 `localhost:3000/@vite/client` 并失败。
- **修复建议**：检查前端 dev 依赖与配置，避免同时启动 Vite 与 Next.js。

## 修复优先级建议

1. **立即修复 P0**：将 `/kaoyan/*` 设为公开浏览，修复 chunk 加载失败，补齐社区列表路由。
2. **短期修复 P1**：替换首页与策略页硬编码、统一空状态与错误提示、补全导师雷达图。
3. **中期优化 P2**：自定义 404、动态筛选选项、清理开发环境配置冲突。
