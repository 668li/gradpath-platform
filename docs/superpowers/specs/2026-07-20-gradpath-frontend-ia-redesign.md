# GradPath 前端信息架构重构设计

**日期**：2026-07-20
**版本**：v1.0
**驱动**：用户反馈"前端内容太多，用户无法准确知道主要功能是什么"
**原则**：按用户任务流组织，不是按功能分类；零功能删除，只重组入口

---

## 1. 第一性原理推导

### 1.1 用户画像与核心痛点

| 维度 | 结论 |
|------|------|
| 用户画像 | 大三/大四/刚毕业 1-2 年的中国大学生，焦虑，信息过载，怕选错 |
| 核心痛点 | 不是"不知道有什么选择"，而是"有太多选择，不知道哪个适合我" |
| 产品本质 | 决策辅助工具，不是信息平台 |
| 护城河 | AI 分析 + 数据验证 + 决策框架，帮用户从"纠结"到"决定" |

### 1.2 用户使用场景（按频率排序）

| 排序 | 场景 | 频率 | 当前入口问题 |
|------|------|------|-------------|
| ① | 迷茫，来诊断一下 | 1 次/用户 | onboarding 做完没"下一步" |
| ② | 查公司/查薪资/查院校 | 高频 | war-room vs 三大方向中心，两个入口冲突 |
| ③ | 做决策（A vs B） | 中频 | 核心护城河，完全藏起来了 |
| ④ | 看自己进度 | 每天 | dashboard 10+ 组件信息爆炸 |
| ⑤ | 问 AI | 每周 | 基本 OK |
| ⑥ | 社区互动 | 每周 | 社区功能分散 3 处 |
| ⑦ | 记录成长 | 偶尔 | 8 个页面散落，0 个有导航入口 |

## 2. 对抗式审查：8 个致命问题

### 问题 1：导航按"功能"分，不是按"任务"分

当前分组：方向 | 工具 | 互动 | 我的

用户第一眼看到"考研/考公/就业"——但用户还没想好方向才来的。应该先诊断方向，再给对应工具。

### 问题 2：核心护城河「决策实验室」完全隐藏

`/decision-lab` 不在导航中。用户需：侧边栏 → 职业规划(空壳页) → 决策实验室。这就像建了金库但把门藏在墙后。

### 问题 3：Dashboard 信息爆炸

10+ 组件（统计卡片 + 决策副驾驶 + 活跃决策 + 审核队列 + 暗知识流 + 记忆事实 + 每周回顾 + 游戏化档案 + 技能雷达 + 连续打卡 + 每日聚焦），用户不知看哪里。

### 问题 4：作战室 vs 三大方向中心，二选一冲突

`/war-room` 聚合三大方向情报但不在导航；`/kaoyan`/`/civil-service`/`/employment` 三个独立中心在导航。同一件事两个入口。

### 问题 5：成长工具 8 个页面散落，0 个有导航入口

技能树、平衡轮、时间线、回顾、成长洞察、成就、生活设计、成长档案——8 个页面，0 个在导航。

### 问题 6：社区功能分 3 处

`/community`（通用）+ `/kaoyan/community`（考研）+ 帖子/问答/标签子页面，用户不知去哪发帖。

### 问题 7：Onboarding 做完就没了，但长期占导航

职业诊断是一次性功能，做完后应变成"诊断报告"。

### 问题 8：管理后台混在主应用

`/admin/*`、`/pipeline/*` 与用户页面在同一路由组。

## 3. 新信息架构

### 3.1 导航结构（7 个入口）

```
┌──────────────────────────────────────────┐
│ GradPath · 职径                           │
│                                           │
│  📊 我的看板      ← 精简版 dashboard       │
│  🎯 决策中心      ← 核心护城河，C位        │
│  🔍 情报中心      ← 合并 war-room + 三大方向 │
│  ────────────────────────────────────────  │
│  🌱 成长档案      ← 合并 8 个成长工具       │
│  💬 社区          ← 合并通用+考研社区       │
│  🤖 AI 对话       ← 原 AI 管家改名          │
│  ────────────────────────────────────────  │
│  👤 个人中心      ← profile + 成就 + 诊断报告 │
└──────────────────────────────────────────┘
```

### 3.2 7 个入口详细设计

#### 3.2.1 我的看板（Dashboard）

**目标**：用户每天看"我当前状态"和"需要我做什么"

**现状改造**：
- **保留**：4 个关键指标卡片（连续打卡、待决策数、待办数、成长等级）
- **保留**：决策副驾驶摘要（待决策事项列表 + 快速操作）
- **保留**：快速入口（查公司、查薪资、写面经、发帖）
- **移除到对应页面**：技能雷达 → 成长档案/技能树；游戏化档案 → 个人中心/成就；每周回顾 → 成长档案/回顾；每日聚焦 → 成长档案；生活平衡轮快照 → 成长档案/平衡轮；暗知识流 → 情报中心；记忆事实 → 个人中心；审核队列 → 管理后台

**布局**：
```
┌────────────────────────────────────────────┐
│ 早上好，张三                          │
│ 你当前有 3 个待决策，2 个待办              │
│                                             │
│ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐      │
│ │ 连续  │ │ 待决策 │ │ 待办  │ │ 成长  │      │
│ │ 7 天  │ │  3 个  │ │ 2 个  │ │ Lv.5  │      │
│ └──────┘ └──────┘ └──────┘ └──────┘      │
│                                             │
│ 需要你决策的事情                          │
│ ┌────────────────────────────────────┐    │
│ │ 字节跳动 vs 腾讯 offer 选择          │    │
│ │ 考研 vs 就业方向选择                  │    │
│ └────────────────────────────────────┘    │
│                                             │
│ 快速入口                                  │
│ [查公司] [查薪资] [写面经] [发帖]          │
└────────────────────────────────────────────┘
```

#### 3.2.2 决策中心（核心护城河）

**合并**：`/decision-lab` + `/decisions` + 决策副驾驶组件

**包含**：
- **待决策 Tab**：所有未完成的决策事项，显示状态、创建时间、紧急度
- **决策记录 Tab**：已完成的历史决策，支持筛选和搜索
- **创建新决策**：进入决策实验室流程（预验尸分析 → 矩阵评分 → 红队挑战 → AI 综合分析）

**设计原则**：决策中心是 GradPath 的护城河，放在导航 C 位（第二个入口），确保每个用户都能发现。

#### 3.2.3 情报中心（合并 war-room + 三大方向）

**合并**：`/war-room` + `/kaoyan` + `/civil-service` + `/employment` + `/interview`

**内部 Tab 结构**：
```
情报中心
├── 📚 考研    ← 原 kaoyan + war-room 考研 Tab
├── 🏛️ 考公    ← 原 civil-service + war-room 考公 Tab
├── 💼 就业    ← 原 employment + war-room 求职 Tab
├── 💰 薪资    ← 原 employment 薪资 Tab
└── 📝 面经    ← 原 interview + war-room 面经 Tab
```

**每个方向 Tab 内部统一布局**：
```
┌──────────────────────────────────────────────┐
│ 考研                                        │
│                                             │
│ [院校情报] [院校对比] [录取预测] [导师情报]  │  ← 子功能快速入口
│ [暗知识] [备考策略]                          │
│                                             │
│ ┌────────────────────────────────────────┐  │
│ │ 推荐院校（基于你的诊断报告）            │  │
│ │ 1. 清华大学 · 计算机 · 匹配度 92%     │  │
│ │ 2. 北京大学 · 计算机 · 匹配度 88%     │  │
│ └────────────────────────────────────────┘  │
│                                             │
│ 资讯列表（最近更新的暗知识/面经）           │
└──────────────────────────────────────────────┘
```

**效果**：用户只需要知道"我要查考研方向"，点进情报中心 → 选考研 Tab → 看信息。不需要在 war-room 和 kaoyan 之间纠结。

#### 3.2.4 成长档案

**合并**：`/growth` + `/skills` + `/life-wheel` + `/life-design` + `/timeline` + `/retrospectives` + `/insights` + `/achievements` + `/growth-patterns`

**内部 Tab 结构**：
```
成长档案
├── 📊 概览      ← 关键指标（技能雷达、平衡轮快照浓缩）
├── 🎯 技能树    ← 原 skills
├── ⚖️ 平衡轮    ← 原 life-wheel（含 intro/assessment/result 三步）
├── 📅 时间线    ← 原 timeline
├── 📝 回顾      ← 原 retrospectives
└── 🏆 成就      ← 原 achievements（含等级+徽章）
```

**设计原则**：8 个入口 → 1 个入口 + 6 个 Tab，用户只需记住"成长相关的东西都在成长档案里"。

#### 3.2.5 社区

**合并**：`/community` + `/kaoyan/community` + `/outcome-report/landing-wall`

**内部 Tab 结构**：
```
社区
├── 🌐 全部      ← 通用 feed（原 /community）
├── 📚 考研专区  ← 原 kaoyan/community（帖子/问答/标签）
├── 🏛️ 考公专区
├── 💼 就业专区
└── 🏆 上岸墙    ← 原 outcome-report/landing-wall
```

#### 3.2.6 AI 对话

**改名**：从"AI管家"改为"AI 对话"

**不变**：路径 `/ai-butler`、功能、交互都不变，仅导航入口名改得更直白

#### 3.2.7 个人中心

**合并**：`/profile` + `/achievements` + `/onboarding`（完成后的诊断报告）

**包含**：
- 个人资料（原 profile）
- 诊断报告（原 onboarding，做完后自动生成为报告）
- 成就与徽章
- 通知设置
- 法律文件链接

## 5. 实施计划

### 阶段 1：导航重构（核心改动）

**文件修改**：
1. `components/nav.tsx` — 重新组织导航入口，7 个入口 + 新分组
2. `app/(app)/layout.tsx` — 调整布局，确保新导航兼容

**注意事项**：
- 移除的导航入口不能导致 404（URL 仍然有效）
- 所有旧的 URL 路径保持不变，仅改变导航入口指向
- `redirect` 策略：部分旧入口可设 301 跳转到新位置

### 阶段 2：Dashboard 精简

**文件修改**：
1. `app/(app)/dashboard/page.tsx` — 精简组件，移除 6 个冗余组件到对应页面
2. 被移除的组件各自对应的页面不需要改动（它们本来就存在）

### 阶段 3：情报中心落地

**文件修改**：
1. 新建 `app/(app)/intel/` 目录（或使用现有路径聚合）
2. `app/(app)/intel/page.tsx` — 情报中心主页面，Tab 切换
3. 每个 Tab 内容：从现有 `war-room`、`kaoyan`、`civil-service`、`employment` 提取对应组件

**与原页面的关系**：
- 原有页面路径不变，功能不变
- 情报中心只是"聚合视图"，不替代原有页面
- 用户可以通过导航直接进情报中心，也可以直接访问原有 URL

### 阶段 4：成长档案落地

**文件修改**：
1. 新建 `app/(app)/growth/`（或增强现有 growth 页面）
2. `app/(app)/growth/page.tsx` — 成长档案主页面，Tab 切换
3. 每个 Tab 内容：从现有 `skills`、`life-wheel`、`timeline`、`retrospectives` 等提取对应组件

### 阶段 5：社区合并

**文件修改**：
1. `app/(app)/community/page.tsx` — 添加 Tab 切换，合并考研社区
2. `app/(app)/community/kaoyan/` — 原 kaoyan/community 内容迁移

### 阶段 6：个人中心增强
