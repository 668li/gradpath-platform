## 一、当前定位：Stage 2.2~2.3

按照五阶段平台演化模型：

| 阶段 | 特征 | 判断标准 | 代表 |
|------|------|---------|------|
| Stage 1 单点工具 | 一次性使用，无留存 | 用完即走 | ChatGPT 简历优化 |
| Stage 2 数据层 | 数据成为资产，新用户受益于历史数据 | 数据复利 | LinkedIn, Boss 直聘 |
| Stage 3 流程嵌入 | 平台可**自动验证用户是否执行** | 行为被客观记录（GitHub提交/刷题/投递记录） | Notion, 粉笔 |
| Stage 4 网络效应 | 用户产生内容影响其他用户 | UGC 反哺平台 | 小红书, 知乎 |
| Stage 5 生态协同 | 第三方围绕平台生长 | 开发者/企业基于平台 API 构建 | 微信, Salesforce |

**GradPath 当前：Stage 2.2~2.3**

已有数据资产，但 AI 生成的是"执行建议"而非"已验证的执行结果"。Stage 3 的判定标准是**平台能否自动验证用户是否执行了行动**（如接入 GitHub 提交记录、牛客刷题记录、BOSS 投递记录等客观数据源），现阶段尚未达到。

用户路径目前是：注册 → 测评 → 看报告 → 离开。目标是：打开 → 看今日行动 → 执行 → 反馈 → 调整计划 → 明天继续。

**12 个月核心目标：从 Stage 2.2~2.3 进入 Stage 3（流程嵌入）。**

## 二、北极星指标：Weighted Weekly Action Score

**不追逐 DAU/MAU。只追逐加权行动完成分。**

```
用户 A
  今日任务：
  □ 投递 3 份简历（权重 20 分）
  □ 联系学长（权重 10 分）
  □ 做 2 道行测（权重 5 分）
  完成率：67%  →  加权得分：30/35 = 85.7%
```

行动需按重要性加权，防止"刷低价值任务"优化指标：

| 行为 | 分值 |
|------|------|
| 阅读文章 | 1 |
| 完成课程 | 5 |
| 简历修改 | 10 |
| 模拟面试 | 15 |
| 实际投递 | 20 |
| 拿到 Offer | 100 |

核心指标：
- **Weighted Weekly Action Score**（加权每周行动得分）
- **Action Completion Rate**（行动完成率，加权后）
- **Streak Days**（连续行动天数）

所有功能迭代围绕"提升加权行动完成率"而非"提升聊天次数"。

## 三、三赛道数据飞轮：GradPath 最强护城河

### 现状：行业断裂

考研平台只懂考研，考公平台只懂考公，招聘平台只懂就业。三者之间数据不互通。

### 真实用户路径是循环的

```
大一 → 考虑考研 → 发现投入产出低 → 转考公 → 发现岗位少 → 转就业
→ 工作两年 → 再考研 → 循环
```

### 飞轮设计（数据流转，非用户流转）

用户生命周期极长（大三 → 毕业 → 3 年后考公 → 5 年后考研），不应设计成"用户 A 流转到用户 B"的互联网飞轮。应设计成**数据流转**：用户 A 产生的数据使未来用户 B 受益。

**飞轮 1：考研数据 → 就业受益**
- 考研用户产生：专业选择数据、学习数据、目标院校数据 → 进入数据库
- 就业用户查询：同专业考研 vs 直接就业的薪资对比、ROI 分析
- AI 输出："计算机本科，过去 10 万人数据，考研后薪资仅提升 18%，建议优先就业积累经验"

**飞轮 2：考公失败数据 → 转轨推荐**
- 考公用户产生：岗位选择、失败原因、转轨路径 → 进入数据库
- 系统发现：同类人群 72% 最终进入国企，AI 自动推荐转向

**飞轮 3：就业后数据 → 深造触发**
- 工作 2 年用户产生：晋升瓶颈、薪资停滞数据 → 触发"是否考虑深造"建议
- 用历史数据计算：硕士收入提升 42%，ROI 评估

### 最终：数据生命周期 10 年+

```
18 岁用户数据 → 考研数据 → 考公数据 → 就业数据 → 职业成长数据 → 再教育数据
```

数据飞轮一旦形成，后来的通用大模型即使更聪明，也无法复制这些时间积累的结果数据。

## 四、商业化路径：Outcome Funnel

**不是卖功能，是卖过程。** 教育/就业/职业规划在中国属于强监管行业，切勿承诺"保证上岸/保证 Offer"——售过程完成度，不售结果承诺。

| 层级 | 定价 | 提供内容 | 目标 |
|------|------|---------|------|
| 免费 | ¥0 | AI 规划、AI 测评、院校查询 | 获客 |
| 会员 | ¥99-299/年 | 行动计划、决策中心、暗知识库全量、成长档案 | 留存 |
| 冲刺包 | ¥199-499 | 简历优化包 / 面试冲刺包 / 调剂包 / 公考冲刺包（单次购买） | 低门槛付费 |
| AI 教练 | ¥999-2999/期 | 90 天考研/考公/求职冲刺（每日任务 + 每周复盘 + AI 督学） | 卖过程 |
| 结果付费 | 完成 100 小时备考计划后 / 完成求职冲刺计划后 | 为过程完成度付费，非结果承诺 | 利润最高 |

**关键合规原则**：
- ❌ 错误："上岸收费" / "保证 Offer" / "保证录取"
- ✅ 正确："完成 100 小时备考计划收费" / "完成求职冲刺计划收费"

借鉴粉笔和中公：**用户买的不是课程，是"有人逼我学"**。GradPath 应做 AI 陪跑官。

参考数据：
- 中公教育"大学四年护航计划"：客户 LTV 提升至传统业务 4.2 倍
- 粉笔 AI 刷题系统班：上线 2 个月销售额 1400 万元，4 万付费学员

## 五、技术护城河：只有 GradPath 能做的四件事

### 壁垒 1：决策结果数据库

```
用户画像 + 决策 A → 结果 A
用户画像 + 决策 B → 结果 B
```

几年后形成 100 万职业决策样本。**这是无法购买的，只能靠时间积累。**

### 壁垒 2：暗知识图谱（含 Decay 机制）

导师脾气、实验室风格、导师放养率、考公真实竞争度、企业隐形门槛 —— 这些公开数据没有，大模型不知道，只有 GradPath 有。

**每条暗知识必须记录**：来源、更新时间、验证次数、置信度。引入自动衰减：2 年未验证的条目自动降权，5 年未验证的标记为"可能已过时"。避免 10 万条目变 10 万条垃圾数据。

### 壁垒 3：行动执行数据

系统发现：每天 7 点推送完成率 18%，晚上 9 点推送完成率 47%。

不断优化，最终形成：**什么人、什么时间、什么任务、最容易完成**。这个价值远超知识库。

### 壁垒 4：Execution Integration Layer（Q4 目标）

MCP 是协议，协议本身无法形成壁垒——任何人都能接入。**真正的壁垒是连接了什么、以及连接后积累的执行数据。**

从 Advice Agent 变成 Execution Agent：
- 求职：连接 Boss/智联/拉勾，自动投递并记录投递结果
- 考研：连接研招网，自动报名提醒 + 分数线追踪
- 考公：连接国考/省考公告，自动岗位匹配

核心壁垒不是协议，而是**连接了这些平台后沉淀的"执行-结果"对照数据**。例如"投递了 30 家 vs 拿到 3 个面试"的转化率，是通用大模型永远无法计算的。

### 架构原则：Core + Domain 解耦

三赛道（考研/考公/就业）看似类似，实际差异巨大。避免后期重构，采用分层架构：

- **Core（共享层）**：用户、行动、成长档案、复盘、决策中心、通知
- **Domain（赛道层）**：考研、考公、就业 → 各自独立服务，通过 Core 的 API 通信

未来新增赛道（如留学、创业）只需新增 Domain 层，Core 不动。

## 六、12 个月路线图

### Q1（2026 年 7-9 月）：流程嵌入——让用户每天打开

**目标行动完成率：30%+**

- 今日行动系统强化：每日 3 个可执行动作，来源标签可视化
- 连续打卡体系：7 天 / 30 天 / 90 天成就徽章
- AI 复盘自动化：每周自动生成复盘草稿（基于本周行动数据）
- 行动完成率看板（dashboard 新增）

### Q2（2026 年 10-12 月）：数据飞轮——让数据产生复利

**目标：积累 10 万+ 决策样本**

- 决策结果追踪：用户做出决策后，定期回访实际结果
- 用户路径数据库：记录用户从考研→考公→就业的完整路径
- 失败案例数据库：失败的决策也是宝贵数据（"为什么没考上"）
- 三赛道交叉推荐：跨赛道决策建议（"考研 ROI 低于直接就业"）

### Q3（2027 年 1-3 月）：商业化——从免费到付费

**目标：月活付费转化率 3%+**

- AI 教练 MVP：90 天冲刺计划（考研/考公/求职三赛道）
- 陪跑计划：每日任务推送 + 每周 AI 复盘 + 督学提醒
- Outcome 收费测试：上岸后付费 / 拿 Offer 后付费
- 企业端 B2B 探索：高校合作（如 Kinobi AI 的 1:1200 师生比解决方案）

### Q4（2027 年 4-6 月）：Agent 化——从给建议到帮执行

**目标：自动化行动验证率 50%+**

- Execution Integration：连接招聘平台/研招网/公考公告，AI 自动匹配岗位并投递
- 自动提醒：考研报名/考公公告/秋招季关键时间节点自动推送
- 自动跟踪：院校分数线变化、岗位竞争度变化实时更新
- 执行结果验证：接入 GitHub/牛客/学习平台等客观数据源，自动验证用户是否完成行动

### Q4 目标：进入 Stage 3

核心标志：**平台能自动验证用户是否执行了行动**（而非仅靠用户自行打卡）。

## 七、核心判断：定位升级（内外有别）

**内部战略定位**：18-35 岁中国青年的人生决策操作系统（Life Operating System）。考研、考公、就业只是第一批场景，未来可扩展婚恋决策、房产决策、城市选择、投资决策。

**外部市场定位**：AI 考研考公就业规划平台。用户搜索"考研规划"而非"人生操作系统"，对外简洁、对内宏大。

**决策安全原则**：所有 AI 输出必须保留多方案 + 证据链 + 不确定性评分，绝不输出"你应该考 XX"，而是输出"根据当前证据，XX 方案胜率 68%"。

**12 个月最重要的不是继续增加功能，而是完成一次定位升级：从"AI 职业规划工具"升级为"AI 行动执行平台"。**

建议已经几乎被大模型免费化了。真正稀缺的是：**让用户持续行动，并沉淀行动结果数据。** 这部分数据飞轮一旦形成，后来的通用大模型即使更聪明，也无法复制。
