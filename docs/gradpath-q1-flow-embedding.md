# GradPath Q1 流程嵌入 — 实施计划

> 基于 Kimi 搜索 + DeepSeek 推理
> 目标：让用户每天打开 GradPath，从"偶尔访问"到"每日习惯"

## 一、Streak 连续打卡体系

### 天数里程碑
- 3天("破冰者") → 7天("习惯雏形") → 14天("双周韧性") → 30天("赛道坚守者") → 60天 → 100天("长期主义")
- 每个节点解锁对应能力：7天解锁AI行为分析，14天解锁AI深度校准，30天颁发数字徽章

### 三赛道适配
- 考研：每日有效专注≥2h
- 考公：每日1套模块训练+错题归因
- 就业：每日投递/人脉/技能学习（三选一）

### 断签处理（聚焦"进度仍在"）
- 每月3次"免罪卡"：断签自动消耗，Streak数字不变，显示"冻结期"
- 免罪卡耗尽后：Streak不归零，置灰"暂停增长"，总活跃天数永远不减
- 补签：完成一次15分钟微复习

### 实施
- 前端：Dashboard 顶部加 StreakBar 组件（天数+进度条+下个里程碑）
- 后端：新增 `/api/streaks/checkin` `/api/streaks/stats`（已有 `/api/streaks/stats` 返回 `StreakStats`）

## 二、AI 周复盘自动生成

### 四层模板
1. 数据快照：本周投入Xh，峰值日周X，完成核心动作A/B/C
2. 对比诊断：vs上周变化，vs同行Top20%
3. 洞察归因：AI推测效率波动原因（如周四频繁切换模块），目标锚定提醒
4. 行动指令：下周建议（周一/三/五攻坚日），3个确认待办

### 可解释性
- 每个洞察旁设"AI依据"气泡（引用具体行为日志）
- 用户可拖拽修改行动计划，AI记录偏好

### 实施
- 前端：新建 `/retrospectives/weekly` 页面（独立于手动复盘）
- 后端：新增 `/api/retrospectives/weekly` 端点（基于本周行为日志自动生成）

## 三、看板"自主驾驶舱"（消灭监控感）

### 设计原则
- 颜色：靛蓝+暖灰主色调，禁用大面积红/橙警告色
- 技能树迷雾：已掌握=发光节点，未掌握=迷雾区（非红色警告）
- 微成就：任务完成时粒子特效+节点点亮
- 撤销权：完成按钮5分钟内可撤销
- 休整日：允许手动标记"今日休整"（不计入缺勤）

### 同赛道能量圈
- 匿名显示"本周超越自己过去3个月80%的周次"
- 非排名竞争，而是自我对比

### 实施
- 修改 `frontend/app/(app)/dashboard/page.tsx`：StreakBar + 技能树迷雾 + 能量圈
- 新增组件：`components/streak-bar.tsx`, `components/skill-tree-fog.tsx`

## 四、推送策略

### 4 臂老虎机
- 臂A（进度）：刚完成任务 → "距离下一徽章仅剩2天"
- 臂B（社交）：浏览过社区 → "你的赛道今日有X人已打卡"
- 臂C（危机）：连续3天低活跃 → "今天只花3分钟守住进度"
- 臂D（洞察）：深度用户 → "AI发现你周一效率奇高，试试把最难任务挪到周一"

### 推送规则
- 时间：用户个性化活跃前30分钟 + 周日晚上 + 月初
- 频次：每周2次，绝不超3次
- 文案：≤10字，带动作锚点

## 五、每日行动闭环

```
早晨（推送触发）→ 打开看板 → 3件核心待办 → Streak数字
日间（碎片执行）→ 微成就反馈 → 技能树点亮
晚间（自然打开）→ Streak更新 → 冻结卡/轻任务
周日（固定仪式）→ AI周复盘 → 编辑下周计划 → 锚定
```

## 本轮实施范围

1. **Dashboard 增强**：StreakBar + 同赛道能量圈 + 技能树进度

3. **每日闭环组件**：今日行动卡片增强（加预估耗时+赛道标签）
4. **后端支撑**：Streak checkin API + Weekly review API
